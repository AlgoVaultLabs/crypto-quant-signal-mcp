=== PARTIAL MITIGATION (caps per-IP, NOT in aggregate) ===
recordSymbolMiss runs AFTER quotaGate() (equity-tool-formatters.ts:108 -> trackCall).
Free-tier counter keys on `free:${getRequestIpHash() || 'anon'}` (license.ts:392) with
getMonthlyQuota('free') = 100 (license.ts:331). So ONE IP-hash can write at most ~100
miss-rows/month. BUT:
  (a) NO dedup => 100 rows of identical garbage per IP/month still bloat the table;
  (b) IP rotation / a distributed caller set => each gets its own fresh 100-row budget;
  (c) the `'anon'` fallback bucket (no IP hash present) is a single shared budget but still
      uncapped across the population;
  (d) append-only with no retention => bloat is monotonic and permanent.
=> Aggregate growth is unbounded; per-IP quota only rate-limits a single source. MEDIUM DoS.

=== GENERATOR-LEVEL FIX (recommended; DEFERRED to follow-up wave — read-only audit) ===
  1. Add UNIQUE(symbol) + rewrite the insert as an upsert-with-counter:
       INSERT INTO equity_symbol_misses (symbol, raw_input, hit_count, requested_at)
       VALUES ($1,$2,1,now())
       ON CONFLICT (symbol) DO UPDATE SET hit_count = equity_symbol_misses.hit_count + 1,
                                          last_requested_at = now();
     -> O(distinct-symbols) rows instead of O(requests). Preserves the demand signal (count is
        actually MORE useful than N duplicate rows). This is the data-quality root-cause fix.
  2. Plus a monthly retention/VACUUM cron (CLAUDE.md append-only-table maintenance) as defense-in-depth.
