### What this actually means

The MCP package ships with **two transports** (src/index.ts:170 stdio mode, src/index.ts:192 HTTP mode). An agent that installs `crypto-quant-signal-mcp` via `npx` by default runs stdio, which connects to the local Node process — **no network hop to api.algovault.com**. The three tools work entirely offline from Hetzner's perspective: they pull candle data directly from Hyperliquid/Binance/Bybit/OKX/Bitget public REST, compute indicators in-process, and return signals. **There is no server-side revenue pressure point on a stdio install.**

So the 1,269 → 18 cliff is **architectural**: 99.2% of NPM installers run stdio and never generate a billable event, a retention signal, or a quota-cap event because they don't need our server for the answer. This is the single highest-leverage finding in this snapshot: **the free, local, zero-server-dependency stdio distribution is eating our funnel**. Every downstream growth tactic — landing page, pricing, cap-tightening — is worthless until activation is moved off stdio and onto the remote API.
